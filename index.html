Informações sobre o Método Delphi e os principais motivos da automatização desse processo

O método Delphi é utilizado há mais de sessenta anos e procura um consenso entre diversos especialistas a respeito de conhecimentos futuros. O Delphi – ou Delfos – é utilizado em inúmeras áreas de conhecimento. Em 1999, surgiu a ideia de aliar o Delphi tradicional à internet, surgindo o Delphi Eletrônico. Porém, a ferramenta não é de fácil acesso. Muitos pesquisadores estão ingressando nesta área de pesquisa, mas há um grande problema: com a falta de ferramentas específicas a pesquisa pode demorar muito. Diante do exposto, há uma necessidade de investimento tecnológico nesta área, pois tais pesquisas demandam tempo por conta de serem realizadas em vários ciclos, resultando em um grande número de dados para serem analisados. A maior dificuldade do método ocorre quando há um grande número de especialistas. Neste sentido é necessário agilizar o tempo da pesquisa, já que ainda não existem muitas opções de softwares gratuitos que foquem, especificamente, nesta técnica de pesquisa. O software Delfos foi proposto para suprir esta necessidade de pesquisadores que utilizem o método Delphi em suas pesquisas. Por ser um software resultante da parceria entre o Instituto Federal de Educação, Ciência e Tecnologia de Rondônia (IFRO) – Campus Ji-Paraná e a Universidade Federal de Rondônia (UNIR) – Campus Ji-Paraná e ambos serem de domínio público, o software será livre e terá o código aberto. Conclui-se que, com tamanha demanda, a criação de um software representa um enorme avanço em relação à velocidade da aplicação do método.

Mais informações sobre o projeto e seu andamento

O presente projeto foi conduzido no Instituto Federal de Educação, Ciência e Tecnologia de Rondônia (IFRO), Campus Ji-Paraná, em parceria com a Universidade Federal de Rondônia (UNIR). A construção do software teve início em Abril (2015). Nas semanas seguintes, dois dias por semana, a equipe se reuniu com o docente do curso de Estatística UNIR (Campus Ji-Paraná), Nério Cardoso, para conhecer mais sobre as necessidades do produto e a área que ele atenderia. Após estar por dentro dos requisitos do software a equipe dedicou-se ao estudo dos cálculos e ao início da criação do Documento de Requisitos. O software será desenvolvido para a automatização do processo de pesquisa. Ele executará cálculos estatísticos com base nas respostas dadas pelos especialistas ao questionário. Este questionário será realizado pelo pesquisador, da maneira que melhor lhe convém, no próprio sistema. Depois de criado o questionário, este será enviado aos especialistas que, por sua vez, responderão os questionários e encaminharão novamente ao pesquisador. Quando o pesquisador receber todos os questionários enviados, o software irá realizar cálculos com base nas respostas. O ciclo se repetirá até que os resultados estejam em consenso. Como os formulários serão gerenciados pelo sistema, os pesquisadores poderão alterá-los para atender as suas necessidades. Os dados coletados pelo pesquisador serão apresentados pelo software de forma sucinta e clara, em forma de gráficos e tabelas, para melhor compreensão por parte do pesquisador. Os especialistas também terão acesso a alguns resultados, no entanto haverá limitações que serão impostas pelo pesquisador. O software terá uma interface simples facilitando a utilização para os leigos em informática. Em sua versão inicial o software será nomeado de Delfos 1.0. O sistema facilitará, tanto em pesquisas qualitativas, quanto em pesquisas quantitativas, pois oferecerá rapidez e precisão no processo e na apuração das pesquisas. O software será construindo usando a linguagem Java para desktops, utilizando-se da ferramenta case Eclipse Luna, os plug-ins WindowBuilder e a framework do WordPress assim como seus plug-ins para o desenvolvimento da aplicação. O Sistema de Gerenciamento de Banco de Dados (SGDB) utilizado na modelagem do banco de dados utilizado pelo software será o MySQL Workbench 6.2 e o phpMyAdmin. Por conta de o software ser construído na linguagem Java, ele poderá ser executado em qualquer sistema operacional que suporte a Java Runtime Environment ou Ambiente de 108 Tempo de Execução Java (JRE). O software funcionará na arquitetura de Cliente-Servidor, onde o banco de dados e os arquivos do sistema estarão presentes no servidor e as máquinas clientes acessarão esses dados. Fica a critério do administrador de redes da empresa como irá funcionar o acesso ao software (Terminal Server, atalhos, etc.). Para o envio de questionários ao especialista, utilizar-se-á um computador responsável pelo banco de dados (local ou server), este por usa vez possuirá um servidor web configurado, podendo ser o Apache, PHP e MySQL instalados pelo pacote Xampp Open Source, devido aos seus níveis de praticidade e integridade do manuseio de informações.

Ideias futuras

O sistema passará por várias atualizações, tendo em vista o cenário pelo qual a área estiver passando. Por ser um software simples e de acesso por internet, espera-se que muitos pesquisadores sejam beneficiados com esta aplicação. O Brasil ainda passa por um processo de desenvolvimento e ainda existem muitas áreas pouco ou inexploradas pela tecnologia, o método de pesquisa Delphi é uma destas áreas. O software Delfos demonstra-se uma ferramenta capaz de armazenar dados de forma segura e executar os cálculos proposto no método Delphi. A implantação e divulgação do software serão de forma abrangente para que o maior número de pesquisadores tenha contato com a ferramenta. O trabalho visa contribuir para com os projetos de pesquisa de acadêmicos dando a eles o conhecimento sobre esta ferramenta que os ajudará muito, o software Delfos. Com incentivos e iniciativas muitos outros softwares poderão ser criados, vindo a beneficiar esta área que é tão importante, a Estatística.
